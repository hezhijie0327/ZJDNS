🔴 Critical 发现（5 项，必须立即修复）

C1. defer 在 for 循环中 → 无限 defer 栈 + 池泄漏

文件: server/client/pool/tcp.go:174
for {
body := bodyBuf[:msgLen]
defer bufpool.DefaultBufferPool.Put(bodyBuf) // ❌ defer 在循环中
}
defer 在外层函数返回前不执行，导致长期 TCP/DoT 连接 defer 栈无限增长，且池缓冲区永不归还。修复: 移除 defer，改用直接 Put 调用。

C2. probeSlice O(n²) Debug 日志

文件: internal/latency/prober.go:211-221
for _, c := range sorted {
for _, r := range results { // O(n²) 嵌套循环
26 个根服务器产生 ~2700 次 extractIP 调用。修复: 将 IP 预先关联到排序结果，单次 O(n) 遍历。

C3. 死代码 — IsEnabled() 始终返回 true

文件: server/security/dnssec_crypto.go:100-102
func (cv \*CryptoValidator) IsEnabled() bool { return cv != nil } — 无调用者。修复: 删除。

C4. 死代码 — ProbeAddrs 45 行未被调用

文件: internal/latency/prober.go:93-138
公开方法但无生产代码调用。修复: 删除。

C5. 死代码 — ShuffleIPs 未被调用 + 注释错误

文件: internal/latency/prober.go:251-262
声称使用 crypto/rand 实际使用 math/rand/v2。修复: 删除。
